# MANUAL DE USUARIO  
## Sistema de Simulación de Algoritmos de Ordenamiento
---

# 1. Introducción
Este sistema permite al usuario simular y comparar diferentes algoritmos de ordenamiento:

- Bubble Sort  
- Shell Sort  
- Quick Sort  

El objetivo es visualizar el proceso de ordenamiento y analizar su rendimiento.
---

#  2. Requisitos del sistema
- Sistema operativo Windows / Linux / Mac  
- Java JDK instalado  
- IDE o terminal de ejecución (NetBeans, IntelliJ, VS Code)
---

#  3. Ejecución del sistema
1. Abrir el proyecto en el IDE  
2. Ejecutar la clase principal (`Main.java`)  
3. Se mostrará el menú principal del sistema  
---

#  4. Uso del sistema
## 4.1 Ingreso de datos
El usuario puede ingresar datos de dos formas:

- Manualmente por consola  
- Cargando un archivo `.txt`

Ejemplo de archivo: 

10 5 3 8 2 7

---

## 4.2 Selección de algoritmo
El sistema muestra un menú como:
1. Bubble Sort
2. Shell Sort
3. Quick Sort
El usuario ingresa el número del algoritmo.
---

## 4.3 Ejecución del algoritmo
El sistema:
- Ordena los datos ingresados  
- Muestra el proceso paso a paso  
- Muestra el resultado final  
---

## 4.4 Generación de reportes
Se genera un reporte con:
- Tiempo de ejecución  
- Número de comparaciones  
- Número de intercambios  
---

# 5. Ejemplo de uso

## Entrada: 
[8, 3, 1, 5]

## Algoritmo seleccionado:
Bubble Sort

## Salida: 
[1, 3, 5, 8]
---

# 6. Errores comunes
- Ingresar letras en lugar de números  
- Archivo TXT con formato incorrecto  
- No seleccionar un algoritmo  
---

# 7. Conclusión

El sistema permite comprender el comportamiento de algoritmos de ordenamiento mediante simulación y análisis de rendimiento.
